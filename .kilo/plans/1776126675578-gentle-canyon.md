# Plan: Allow Trial → Inactive Status Transition

## Problem

When a trial student decides not to sign up, there's no way to mark them as Inactive. The current status dropdown for trial students only shows "Trial" and "Active" options.

## Current State Machine (Trial)

```
trial → active  (sign up via ActivateStudentDialog)
trial → trial   (stay)
```

## Target State Machine (Trial)

```
trial → active    (sign up via ActivateStudentDialog)
trial → inactive  (didn't sign up, clear membership data)
trial → trial     (stay)
```

## Changes Required

### File: `src/pages/StudentDetail.tsx` (2 edits)

**Edit 1: Add "Inactive" option to Trial dropdown** (~lines 745-749)

Add `<SelectItem value="inactive">Inactive</SelectItem>` inside the `{isTrial && ...}` block:

```tsx
{isTrial && (
  <>
    <SelectItem value="trial">Trial</SelectItem>
    <SelectItem value="active">Active</SelectItem>
    <SelectItem value="inactive">Inactive</SelectItem>
  </>
)}
```

**Edit 2: Handle trial→inactive in `updateStatusMutation`** (~lines 566-584)

The existing `"frozen" || "inactive"` branch only sets `membership_status`, but for trial students `status` is still `"trial"`. The display logic prioritizes `isTrial` over `isInactive`, so the UI would still show "Trial".

Fix: when current status is trial and target is inactive, also set `status: "none"` and clear `membership_plan_id`. Add a new branch before the frozen/inactive block:

```typescript
} else if (newStatus === "inactive" && student?.status === "trial") {
  // Trial student didn't sign up — mark as inactive, reset status to "none"
  updates = { status: "none", membership_status: "inactive", membership_plan_id: null };
} else if (newStatus === "frozen" || newStatus === "inactive") {
  // ... existing logic for active students ...
}
```

No other files need changes — the display logic, badge rendering, and Students list already handle the "inactive" state correctly.
