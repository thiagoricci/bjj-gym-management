# Plan: Show Failed Payments in Student Payment History & Dashboard

## Problem

When a payment fails (card declined, insufficient funds, etc.), nothing is recorded in the `payments` table. The student's payment history shows no indication of failure, and there's no visibility on the Dashboard.

## Root Cause

1. **Stripe webhook** has no handler for `invoice.payment_failed` or `payment_intent.payment_failed`.
2. **charge-student** edge function returns an error on failure but doesn't insert a `status: 'failed'` payment row.
3. **PaymentHistory** component renders status as plain text with no visual distinction.
4. **Dashboard** has no section showing recent failed payments.

## Changes

### 1. Database Migration â€” Add `failure_reason` column

**New file:** `supabase/migrations/<timestamp>_add_failure_reason_to_payments.sql`

Add a nullable `failure_reason` text column to `payments` so Stripe error messages (e.g., "card declined", "insufficient funds") can be stored and displayed.

```sql
ALTER TABLE payments ADD COLUMN IF NOT EXISTS failure_reason text;
```

### 2. Stripe Webhook â€” Add `invoice.payment_failed` handler

**File:** `supabase/functions/stripe-webhook/index.ts`

Add a new `else if` branch after line 252 (after the subscription update handler):

- Listen for `invoice.payment_failed`
- Retrieve subscription metadata (`studentId`, `planId`, `organizationId`) from the invoice's subscription
- Look up the plan price from `membership_plans`
- Insert a payment row with `status: 'failed'`, the invoice amount, and `failure_reason` from `invoice.last_finalization_error?.message`
- This captures **recurring billing failures** and **async initial payment failures**

### 3. charge-student â€” Record failed payment on error

**File:** `supabase/functions/charge-student/index.ts`

In the failure branch (lines 253-266, the `else` block), insert a payment record with `status: 'failed'` and `failure_reason` from the payment intent error before returning the error response. This captures **synchronous failures** during the initial charge attempt.

### 5. Dashboard â€” Add "Failed Payments" alert section

**File:** `src/pages/Dashboard.tsx`
